EventBUS的使用

鱼八 · 2020-1-2 · 次阅读

EventBus

有过了解EventBus的同学都知道，EventBus 是一个Android 优化的 publish / subscribe 消息事件总线
简化了应用程序各个组件间、组件与后台线程间的通信。如：Activities Framents Threads Services
如果你之前没有了解过可以点击此处去了解一下。
EventBus is a publish/subscribe event bus for Android and Java.

现在我们知道EventBus是用于处理消息事件的，让组件与组件，组件与服务之间消息的传递变得更加简单。当然在我们学习了解使用EventBus之前，我们首先了解一下常规的事件传递方式。

一、事件的传递

1.常规事件的传递

在过去我们常使用以下的事件传递方式

  • Intent 跳转 + 传参 （局限性大）
  • Handler 通常用来更新主线程的UI，使用不当容易出现内存泄漏
  • Interface 接口，仅限于同一线程中数据交互
  • BroadCastReceiver 有序广播+无序广播
  • AIDL 夸进程通信 阅读较难，维护成本高
  • 其他方式，比如本地存储等

由于这些方式或多或少都存着一定的问题，所以我们现在选择使用了EventBUS，它简化了应用程序内各个组件之间进行通信的复杂度，尤其是碎片之间进行通信的问题，可以避免由于使用广播通信而带来的诸多不便。

###2. EventBus 事件的传递
现在我们使用EventBus来替代这些方式
eg: 比如网络请求，在返回时通过Handler或者BroadCastReceiver 通知更新主线程UI ， 在N个Fragment之间需要通过listener 通信。这些都可以通过EventBus来实现。

在官网中，我们可以看到这样一张图：

Publisher 发布者 —> 使用post方式 发送Event事件 —> 通过EventBus 匹配所有订阅的方法，匹配到以后在发送出去Event —>到 Subscriber 订阅者那里也就是执行了 OnEvent()事件。

了解了EventBus 事件模型后，我们在来看看EventBus的基本使用：

2.1EventBus的基本使用

在使用EventBus之前 我们需要先添加依赖：

implementation 'org.greenrobot:eventbus:3.1.1'
2.1.1 普通事件
  • 首先我们需要先定义发送的事件。
    public static class MessageEvent { /* Additional fields if needed */ }
  • 然后订阅该事件 
    //订阅方法的事件 是 需要 和发送的事件相匹配的。
//例如这里是 MessageEvent 事件，那么只有post MessageEvent事件，该订阅事件才会被执行。
@(3.架构师)  
public void onMessageEvent(MessageEvent event) {/* Do something */};
    • 接着我们在当前注册和反注册
      注册
      @Override
public void onStart() {
 super.onStart();
 EventBus.getDefault().register(this);
}
  • 反注册
    @Override
public void onStop() {
   super.onStop();
   EventBus.getDefault().unregister(this);
}
  • 最后我们在需要发送事件的地方，发送事件
    EventBus.getDefault().post(new MessageEvent());
2.1.2 粘性事件

前面是EventBus最基本的使用。但实际上EventBus还有一个事件是粘性事件，主要作用是用于延时消费 或者未初始化界面。发送事件后，当界面初始化完成后，才会接收到事件。

//订阅方法的事件 是 需要 和发送的事件相匹配的。
//例如这里是 MessageEvent 事件，那么只有post MessageEvent事件，该订阅事件才会被执行。

//这里是粘性事件，在@Subscribe 中 加入 sticky = true 
@Subscribe(threadMode = ThreadMode.MAIN,sticky = true)  
public void onMessageEvent(MessageEvent event) {/* Do something */};

发送粘性事件

 EventBus.getDefault().postSticky(new MessageEvent());
2.1.3 事件的优先级

如果俩个事件同时接收，proprity 数值越大，优先级越高，一般默认为0

@Subscribe(threadMode = ThreadMode.MAIN,priority = 1)  
public void onMessageEvent(MessageEvent1 event) {/* Do something */};

@Subscribe(threadMode = ThreadMode.MAIN,priority = 10)  
public void onMessageEvent(MessageEvent2 event) {/* Do something */};

2.2 正确的使用EventBus

当我们按照以上方式写2个简单的Activity跳转发送事件，实际上是接收不到事件的（因为在A Activity跳转时执行了OnStop()方法，EventBus方法会被反注册，所以实际上订阅方法就不会被执行了。）。所以很多同学都在OnCreate方法里注册，在OnDestroy方法里反注册，再去发送事件，订阅事件就会被调用了。最终实现了EventBus最基本的使用。

  @Override
    protected void onCreate(Bundle savedInstanceState) {
        super.onCreate(savedInstanceState);
        setContentView(R.layout.activity_main);
        EventBus.getDefault().register(this);
    }

@Override
protected void onDestroy() {
    super.onDestroy();
    //这里养成良好的习惯
    if (EventBus.getDefault().isRegistered(this)) {
        EventBus.getDefault().unregister(this);
    }
}
